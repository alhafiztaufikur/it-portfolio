How I Automated Cisco Router Backup with Python & Netmiko

Al Hafiz Taufikur Rohman June 2025 8 min read

The Problem: Manual Backups Are a Liability

Managing 50+ Cisco routers and switches in a university network lab means one wrong change can cascade into a multi-hour outage. The biggest risk? No reliable config backup. Manually SSH-ing into each device, running show running-config, and copy-pasting into a text file is error-prone and simply doesn't scale.

I needed a system that: automatically connects to every device, pulls the running config, detects if anything changed since the last backup, and stores versioned copies in a Git repository — all without touching a single CLI manually.

Tools & Prerequisites

  • Python 3.10+ — the automation engine
  • Netmiko — SSH library for network devices (pip install netmiko)
  • GitPython — Git operations from Python (pip install gitpython)
  • A local Git repository initialized in the backup folder
  • SSH access enabled on all target Cisco devices

Step 1 — Define the Device Inventory

I store the list of devices in a simple Python dictionary. In production, you'd pull this from a YAML file or a CMDB, but a dict works perfectly for a lab environment:

Python
# devices.py — Network Device Inventory DEVICES = [ { "device_type": "cisco_ios", "host": "192.168.1.1", "username": "admin", "password": "SecurePass123", "hostname": "Core-R1", }, { "device_type": "cisco_ios", "host": "192.168.1.2", "username": "admin", "password": "SecurePass123", "hostname": "Dist-SW1", }, # ... add all 50+ devices here ]
Security tip: Never hardcode credentials in production. Use environment variables or a secrets manager like HashiCorp Vault.
os.environ.get("NETWORK_PASSWORD")

Step 2 — Connect and Pull the Running Config

Netmiko handles the SSH handshake and session management. The ConnectHandler context manager automatically closes the connection after the block exits — no leaked sessions.

Python
from netmiko import ConnectHandler from netmiko.exceptions import NetMikoTimeoutException, AuthenticationException import logging def backup_device(device: dict) -> str | None: """Connect to a device and return its running-config as a string.""" try: with ConnectHandler(**device) as conn: conn.enable() # Enter privileged EXEC mode config = conn.send_command("show running-config") return config except NetMikoTimeoutException: logging.error(f"Timeout connecting to {device['host']}") except AuthenticationException: logging.error(f"Auth failed for {device['host']}") return None

Step 3 — Save to File & Detect Drift with Git

After pulling the config, I write it to a file named after the device. Then I use GitPython to stage and commit — Git's own diff engine handles detecting what changed between backups.

Python
from git import Repo import datetime, pathlib BACKUP_DIR = pathlib.Path("./backups") repo = Repo(BACKUP_DIR) def save_and_commit(hostname: str, config: str): filepath = BACKUP_DIR / f"{hostname}.cfg" filepath.write_text(config, encoding="utf-8") # Stage the file repo.index.add([str(filepath.relative_to(BACKUP_DIR))]) # Only commit if something actually changed if repo.is_dirty(): timestamp = datetime.datetime.now().strftime("%Y-%m-%d %H:%M") repo.index.commit(f"backup: {hostname} @ {timestamp}") print(f"[+] Config drift detected — committed backup for {hostname}") else: print(f"[=] No changes on {hostname}")

Step 4 — Run in Parallel for All Devices

Running backups sequentially across 50+ devices would take too long. I use Python's ThreadPoolExecutor to connect to multiple devices concurrently:

Python
from concurrent.futures import ThreadPoolExecutor from devices import DEVICES def run_backup(device): config = backup_device(device) if config: save_and_commit(device["hostname"], config) if __name__ == "__main__": print(f"Starting backup for {len(DEVICES)} devices...") with ThreadPoolExecutor(max_workers=10) as executor: executor.map(run_backup, DEVICES) print("Backup complete.")

Step 5 — Schedule with Cron (Linux) or Task Scheduler

Run the backup automatically every night at 2 AM:

Cron (Linux)
# crontab -e 0 2 * * * /usr/bin/python3 /opt/network-backup/backup.py >> /var/log/netbackup.log 2>&1

Results & Lessons Learned

  • ✅ Full backup of 50+ devices completes in under 90 seconds with 10 parallel workers
  • ✅ Git history provides a clear audit trail of every configuration change
  • ✅ Configuration drift is detected immediately on the next scheduled run
  • ⚠️ Some older Cisco IOS versions require setting global_delay_factor=2 in Netmiko to avoid timeout issues
  • ⚠️ Enable logging synchronous on devices to prevent console messages from breaking the SSH output
